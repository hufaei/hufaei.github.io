# 番剧观看档案站架构设计说明

## 1. 项目定位

本项目是部署在 GitHub Pages 上的全静态番剧观看档案站。

它用于记录：

- 看过或正在看的番剧
- 每部番剧的观看进度
- 系列、季度、剧场版、OVA 之间的层级关系
- 外部公开评分与番剧基础资料

它不用于记录个人短评，也不做个人评分。

## 2. 技术选型

| 模块 | 方案 |
| --- | --- |
| 静态站生成 | Astro |
| 语言 | TypeScript |
| 部署 | GitHub Pages |
| 本地内容 | JSON 文件 |
| 主数据源 | Bangumi API |
| 补充数据源 | AniList GraphQL API |
| 测试 | Vitest + Playwright |

选择 Astro 的原因是它适合生成全静态页面，默认前端运行时代码少，适合 GitHub Pages 这种无服务器环境。

## 3. 总体架构

```text
用户维护的番剧列表
        │
        ▼
content/*.json
        │
        ▼
元数据同步脚本
Bangumi + AniList
        │
        ▼
本地缓存与归一化数据
data/generated/*.json
        │
        ▼
Astro 静态页面构建
        │
        ▼
GitHub Pages 发布
```

线上页面不依赖实时 API。番剧资料、评分、封面、横幅等都在构建期同步并缓存到仓库中。

## 4. 页面结构

| 路由 | 说明 |
| --- | --- |
| `/` | 首页，展示当前在看、最近完成、统计和入口 |
| `/anime/` | 番剧档案列表，支持分页、搜索和筛选 |
| `/anime/:slug/` | 单部番剧详情页 |
| `/series/:slug/` | 系列页，展示该系列下所有季度、剧场版、OVA |
| `/timeline/` | 观看事件时间线 |
| `/tags/:tag/` | 标签归档页 |
| `/search/` | 搜索页 |

系列页用于解决第二季、第三季、剧场版等问题。用户先进入系列，再看到该系列下每一部具体作品。

## 5. 数据模型

### Series

表示一个系列，例如“葬送的芙莉莲”。

包含：

- 系列标题
- 系列 slug
- 子条目列表
- 系列封面代表项
- 展示顺序
- 聚合观看进度

### Anime Entry

表示一部具体可观看条目，例如第一季、第二季、剧场版或 OVA。

包含：

- Bangumi ID
- AniList ID
- 标题
- 集数
- 已看集数
- 状态
- 封面
- 横幅
- 外部评分
- 所属系列

### Watch Event

表示一次观看状态变化。

包含：

- 开始观看
- 更新到第几集
- 已完成
- 暂停
- 搁置
- 重看

这只是结构化记录，不包含短评。

## 6. 进度计算

单部番剧优先使用：

```text
已看集数 / 总集数
```

如果集数不明确，可使用百分比。

系列进度由子条目聚合：

```text
系列已看总集数 / 系列总集数
```

剧场版可视为 `1 / 1`。

## 7. 外部数据源

### Bangumi

Bangumi 是主数据源。

用于获取：

- 中文标题
- 原始标题
- 简介
- 封面
- 集数
- 标签
- 公开评分
- 排名
- 投票数
- 关联条目

页面中展示的公开评分默认使用 Bangumi，并标注来源。

### AniList

AniList 是补充数据源。

用于获取：

- 横幅图
- 封面代表色
- 英文标题
- Romaji 标题
- 别名
- 关系图
- AniList 公开评分和热度

AniList 不覆盖 Bangumi 的中文主数据，只做视觉和国际化补充。

## 8. 搜索与分页

构建时生成轻量搜索索引：

```text
data/generated/search-index.json
```

索引只包含搜索需要的字段：

- 标题
- 别名
- 标签
- 状态
- 年份
- slug
- 进度

长简介和完整详情不进入搜索索引，避免前端加载过大。

列表页使用静态分页，避免一次性渲染大量番剧卡片。

## 9. 多语言与主题

默认状态：

```text
语言：中文
主题：暗色
```

支持：

- 中文
- 日语
- English
- 暗色主题
- 亮色主题

语言和主题选择保存到 `localStorage`。

亮色主题不做普通白底，而是保留灰纸、漫画稿纸和黑线结构，避免与整体二次元暗色风格割裂。

## 10. 性能设计

性能原则：

- 构建期同步元数据，运行时不请求外部 API
- 静态 HTML 优先
- 搜索索引保持小体积
- 图片懒加载
- 首页和详情页主视觉优先加载
- 列表分页
- 前端 JavaScript 只承担搜索、筛选、语言和主题切换

## 11. 部署流程

本地开发：

```bash
npm run dev
```

同步番剧元数据：

```bash
npm run data:update
```

刷新已缓存元数据：

```bash
npm run data:update:refresh
```

验证并构建站点：

```bash
npm run verify
```

发布目标：

```text
https://hufaei.github.io/
```

GitHub Actions 后续负责自动构建并发布到 GitHub Pages。
